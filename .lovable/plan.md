

# План: Исправление позиционирования модального окна апсейла

## Проблема

При клике на название апсейла для открытия детальной карточки:
1. Модалка появляется, но страница остаётся прокручиваемой
2. Пользователю приходится скроллить вверх, чтобы увидеть модалку
3. Видно hero-фото товара позади — это странный UX

## Причина

Модальное окно имеет `fixed inset-0`, но:
- Не блокирует скролл `body`
- Страница за модалкой остаётся интерактивной

## Решение

Добавить блокировку скролла при открытии модалки:

```tsx
useEffect(() => {
  if (detailProduct) {
    // Блокируем скролл body
    document.body.style.overflow = 'hidden';
  } else {
    // Возвращаем скролл
    document.body.style.overflow = '';
  }
  
  // Cleanup при размонтировании
  return () => {
    document.body.style.overflow = '';
  };
}, [detailProduct]);
```

## Дополнительные улучшения

1. **Добавить `overscroll-behavior: contain`** — предотвращает "проваливание" скролла
2. **Центрирование модалки** — убедиться, что `items-center justify-center` работает корректно с `fixed`

## Результат

- Модалка появляется **поверх** текущей позиции экрана
- Страница за модалкой не скроллится
- Пользователю не нужно ничего искать — всё на месте

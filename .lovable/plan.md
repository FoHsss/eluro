
# План: Полное скрытие hero-фото при скролле до footer'а

## Диагностика проблемы

На скриншоте видно:
1. Hero-изображение (ошейник) остаётся **sticky** и "приклеено" к верху экрана
2. Когда контент заканчивается, hero всё ещё виден над footer'ом
3. Background layer (`absolute inset-0`) покрывает только `motion.div`, но **не достигает footer'а**

### Корневая причина

Текущая структура:
```
Layout
├── Header
├── main (pt-[100px])
│   └── ProductPage
│       ├── div.pb-20
│       │   ├── Hero (sticky top-9, z-0) ← остаётся видимым!
│       │   └── Content (z-10, min-h-screen)
│       │       └── absolute bg-background ← не достигает footer
└── Footer ← hero виден между content и footer
```

## Решение

### Идея пользователя

Сделать так, чтобы hero-фото **переставало быть sticky** в определённой точке, и всё, что ниже — было белым фоном.

### Реализация

**Шаг 1: Отключить sticky раньше**

Сейчас sticky отключается при `reviewsInView`, но footer находится **ещё ниже**. Нужно отключить sticky раньше — например, использовать отдельный ref для "границы sticky".

**Шаг 2: Добавить белый "отсечник" внизу контента**

Добавить дополнительный div с белым фоном, который **гарантированно** закрывает hero снизу:

```tsx
{/* White spacer to cover hero before footer */}
{isMobile && (
  <div className="bg-background h-[60vh]" />
)}
```

**Шаг 3: Убрать pb-28 и положиться на spacer**

Вместо `pb-28` на контенте использовать явный spacer, который будет выше hero (55vh) и полностью его закроет.

## Файлы для изменения

| Файл | Изменения |
|------|-----------|
| `src/pages/ProductPage.tsx` | Добавить белый spacer блок в конце контента, чтобы hero был полностью скрыт перед footer'ом |

## Визуальный результат

**До:**
```
┌────────────────┐
│    Hero        │ sticky z-0
├────────────────┤
│   Content      │ z-10
│   ...          │
│ ─ ─ ─ ─ ─ ─ ─ ─│ ← контент кончился
│   [HERO виден] │ ← 55vh hero проглядывает!
├────────────────┤
│    Footer      │
└────────────────┘
```

**После:**
```
┌────────────────┐
│    Hero        │ sticky z-0
├────────────────┤
│   Content      │ z-10
│   ...          │
│                │
│   [Spacer]     │ ← белый блок 60vh
│                │ ← полностью закрывает hero
├────────────────┤
│    Footer      │
└────────────────┘
```

## Технические детали

Высота spacer'а должна быть **больше или равна** высоте hero (55vh), чтобы гарантированно закрыть его при любом скролле. Используем `h-[60vh]` с запасом.

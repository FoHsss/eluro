
# План: Исправление видимости hero-фото под футером

## Проблема

На мобильных устройствах при прокрутке в самый низ страницы:
- Sticky-hero (продуктовое фото) остаётся видимым из-под контента
- Появляется зазор между контентом и футером, через который просвечивает hero

На скриншоте видно: между белым контентом и footer'ом проглядывает часть hero-изображения (ошейник странного ракурса).

## Причина

В `ProductPage.tsx` есть слой для перекрытия sticky-hero:

```tsx
{isMobile && (
  <div className="absolute inset-x-0 top-32 bottom-0 bg-background -z-10" />
)}
```

**Проблемы:**
1. `top-32` оставляет зазор в 128px сверху
2. Контент использует `min-h-screen`, но контент может быть **длиннее** экрана
3. Sticky-hero остаётся "приклеенным" и виден за пределами контента

## Решение

### Шаг 1: Изменить `top-32` на `top-0`

Фоновый слой должен начинаться с самого верха контента:

```tsx
{isMobile && (
  <div className="absolute inset-0 bg-background -z-10" />
)}
```

### Шаг 2: Убрать зависимость от `min-h-screen`

Вместо `min-h-screen` использовать полноценное покрытие с `inset-0`.

### Шаг 3: Добавить отступ снизу через Footer

Если необходимо — убедиться, что Footer имеет собственный `bg-background`, чтобы полностью перекрыть hero (проверил — Footer уже имеет `bg-background`).

## Файлы для изменения

| Файл | Изменения |
|------|-----------|
| `src/pages/ProductPage.tsx` | Изменить `top-32` на `inset-0` для фонового слоя на мобильных |

```
│    Hero        │ ← sticky, z-0
├────────────────┤
│   Content      │ ← z-10, bg с top-32
│                │
│   ...          │
├────────────────┤
│   [ЗАЗОР]      │ ← видно hero!
├────────────────┤
│    Footer      │
└────────────────┘
```

**После:**
```
┌────────────────┐
│    Hero        │ ← sticky, z-0
├────────────────┤
│   Content      │ ← z-10, bg с inset-0
│                │
│   ...          │
│                │ ← белый фон до конца
├────────────────┤
│    Footer      │
└────────────────┘
```
